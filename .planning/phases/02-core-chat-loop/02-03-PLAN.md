     ```html
     <script>
       (function() {
         const mode = localStorage.getItem('mode-watcher-mode');
         if (mode === '"dark"' || (!mode && window.matchMedia('(prefers-color-scheme: dark)').matches)) {
           document.documentElement.classList.add('dark');
         }
       })();
     </script>
     ```
     Note: Check if mode-watcher already injects this. If `ModeWatcher` handles FOUC via its own inline script, skip this step to avoid conflicts. Test by toggling to light mode and hard-refreshing — if no flash, the script is not needed.

  **highlight.js themes for dark/light mode (Pitfall 6):**

  4. In `web/src/app.css`, add highlight.js theme imports using CSS layers so dark mode works:
     - Import `highlight.js/styles/github.css` as the light theme.
     - Import `highlight.js/styles/github-dark-dimmed.css` for dark mode.
     - Use the Tailwind v4 dark variant: wrap the dark theme import in `.dark` scope.
     - Approach: Since CSS @import must be at top level, use this strategy:
       ```css
       /* After the tailwindcss import */
       @import "highlight.js/styles/github.css";
       ```
       Then for dark mode, add a scoped override:
       ```css
       .dark pre code.hljs {
         /* Override with github-dark-dimmed colors */
         background: oklch(0.21 0.006 285.885);
         color: #adbac7;
       }
       ```
       OR: Import both themes and scope them:
       ```css
       @import "highlight.js/styles/github.css" layer(hljs-light);
       @import "highlight.js/styles/github-dark-dimmed.css" layer(hljs-dark);
       ```
       Then conditionally apply via `.dark` class. The exact approach should be whatever works cleanly with Tailwind v4's @import system. Test that code blocks look correct in both modes. If CSS @import layering is problematic, fall back to importing one theme statically and overriding key colors in the `.dark {}` block in app.css.
