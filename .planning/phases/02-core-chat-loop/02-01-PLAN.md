  <name>Task 2: Build SSEClient and markdown utilities</name>
  <files>web/src/lib/chat/SSEClient.ts, web/src/lib/chat/markdown.ts</files>
  <action>
  1. Create `web/src/lib/chat/SSEClient.ts`:

     Export an async function `streamChat(message: string, sessionId: string | undefined, onEvent: (type: string, data: any) => void, signal: AbortSignal): Promise<void>`.

     Implementation (follow Pattern 2 from RESEARCH.md exactly):
     - POST to `/api/chat/stream` with JSON body `{ message, session_id: sessionId }` and the AbortSignal.
     - Check `response.status === 429` BEFORE reading body — throw `new Error('Session is busy')`.
     - Check `!response.ok` — throw `new Error(\`HTTP ${response.status}\`)`.
     - Pipe response.body through `TextDecoderStream()` then `EventSourceParserStream()` (from `eventsource-parser/stream`).
     - Use `for await (const event of stream)` to iterate. On each event: if `signal.aborted` break; otherwise try `onEvent(event.event ?? 'message', JSON.parse(event.data))` with catch to skip malformed events.
     - Do NOT catch the AbortError from fetch — let it propagate so the caller can distinguish abort vs real error.

  2. Create `web/src/lib/chat/markdown.ts`:

     **Final render function** — `renderMarkdown(text: string): Promise<string>`:
     - Guard with `import { browser } from '$app/environment'` — return plain text if not browser.
     - Lazily initialize a `Marked` instance with `markedHighlight` configured for highlight.js (use `new Marked(markedHighlight({...}))` pattern — NOT `marked.use()`).
     - Lazily dynamic-import DOMPurify: `const { default: DOMPurify } = await import('dompurify')`. Cache both in module-level variables.
     - Parse with `marked.parse(text)` then sanitize with `DOMPurify.sanitize(html)`.
     - Configure DOMPurify.sanitize to allow class attributes (needed for hljs classes): `DOMPurify.sanitize(html, { ADD_ATTR: ['class'] })`.

     **Streaming animation function** — `animateStreaming(container: HTMLElement, text: string, onDone: () => void): () => void`:
     - Import `* as smd from 'streaming-markdown'`.
     - Create renderer with `smd.default_renderer(container)`, then parser with `smd.parser(renderer)`.
     - Use setInterval at 16ms, writing 8 characters per tick via `smd.parser_write(parser, chunk)`.
     - When text exhausted: call `smd.parser_end(parser)`, clearInterval, then call `onDone()`.
     - Return a cancel function that: clears the interval. (The caller handles swapping to final marked render on cancel.)

     **Post-animation highlight** — `highlightCodeBlocks(container: HTMLElement): void`:
     - Import hljs. Find all `code[class*="language-"]` elements inside container and call `hljs.highlightElement(el)` on each.
     - Also export this so MessageBubble can call it after streaming animation ends and before the marked swap.
  </action>

<task type="auto">
  <name>Task 3: Build reactive ChatStore</name>
  <files>web/src/lib/chat/ChatStore.svelte.ts</files>
  <action>
  Create `web/src/lib/chat/ChatStore.svelte.ts`:

  Export a `ChatStore` class using Svelte 5 reactive class pattern ($state fields). Export a singleton `chatStore` instance.

  **$state fields:**
  - `messages = $state<Message[]>([])` — all messages in current conversation
  - `status = $state<ChatStatus>('idle')` — current chat state
  - `currentActivity = $state<string>('')` — activity text during loading (e.g., "Using: read_file")
  - `lastError = $state<string>('')` — error message for display
  - `sessionId = $state<string>(crypto.randomUUID())` — current session ID

  **Non-reactive fields:**
  - `abortController: AbortController | null = null` — for cancelling in-flight requests
  - `cancelAnimation: (() => void) | null = null` — for cancelling streaming animation

  **Methods (use arrow functions to avoid `this` binding issues per Pitfall 7):**

  - `sendMessage = async (text: string): Promise<void>` — The core send flow:
    1. If `this.status !== 'idle'` return early (no message queuing per discretion decision).
    2. Set `status = 'loading'`, `currentActivity = 'Connecting...'`, `lastError = ''`.
    3. Push user message to `this.messages` (role: 'user', content: text, id: crypto.randomUUID(), timestamp: Date.now(), isStreaming: false).
    4. Create new AbortController, store as `this.abortController`.
    5. Call `streamChat(text, this.sessionId, onEvent, controller.signal)` in a try/catch.
    6. onEvent handler (define as inner function):
       - `'intent'`: set `currentActivity = \`Mode: ${data.mode}\``
       - `'tool'`: set `currentActivity = \`Using: ${data.tool}\``
       - `'loop'`: set `currentActivity = 'Thinking...'`
       - `'done'`: set `status = 'streaming'`, push assistant message (role: 'assistant', content: data.response, id: crypto.randomUUID(), timestamp: Date.now(), isStreaming: true). Store `data.session_id` if returned.
       - `'error'`: set `status = 'error'`, `lastError = data.error`. Push error message (role: 'error', content: data.error, id: crypto.randomUUID(), timestamp: Date.now(), isStreaming: false).
    7. On catch: if NOT AbortError, set `status = 'error'`, `lastError = e.message`. Push error message to messages.
    8. On finally (only when not streaming): set `abortController = null`.
    Note: status transitions to 'idle' after animation completes (driven by MessageBubble calling markDone).

  - `stop = (): void` — Cancel current operation:
    1. Call `this.abortController?.abort()` and set to null.
    2. Call `this.cancelAnimation?.()` and set to null.
    3. If `this.status === 'streaming'`: mark the last assistant message as `isStreaming = false` (animation stopped, bubble will swap to final render).
    4. Set `status = 'idle'`, `currentActivity = ''`.

  - `retry = async (): Promise<void>` — Retry after error:
    1. Find the last user message in `this.messages` (filter by role === 'user', take last).
    2. Remove trailing error messages from `this.messages` (pop while last message role is 'error').
    3. Set `status = 'idle'`, then call `this.sendMessage(lastUserMessage.content)`.

  - `markDone = (messageId: string): void` — Called by MessageBubble when animation completes:
    1. Find message by id, set `isStreaming = false`.
    2. If `status === 'streaming'`, set `status = 'idle'`, `currentActivity = ''`.

  - `registerAnimationCancel = (cancel: () => void): void` — Store cancel function from animation.

  Export singleton: `export const chatStore = new ChatStore();`
  </action>
