  <name>Task 1: Build MessageBubble and InputBar components</name>
  <files>web/src/lib/components/MessageBubble.svelte, web/src/lib/components/InputBar.svelte</files>
  <action>
  **MessageBubble.svelte** — Renders a single message (user, assistant, or error role).

  Props: `message: Message` (from types.ts via $props()).

  Layout (LOCKED per user decision — bubble style):
  - Outer div: `flex w-full` with `justify-end` for user, `justify-start` for assistant/error.
  - No avatars (LOCKED — alignment alone distinguishes sender).
  - Bubble: `max-w-[80%] rounded-2xl px-4 py-3 text-sm` with:
    - User: `bg-primary text-primary-foreground rounded-br-sm`
    - Assistant: `bg-muted text-foreground rounded-bl-sm`
    - Error: `bg-destructive/10 text-destructive rounded-bl-sm`
  - User messages: render content as `whitespace-pre-wrap` plain text.
  - Error messages: render content text + retry button (calls `chatStore.retry()`).

  Assistant message rendering (two-mode markdown — LOCKED):
  - Use a `div` with `bind:this={markdownContainer}` and class `prose prose-sm dark:prose-invert max-w-none`.
  - On mount ($effect): if `message.isStreaming === true`, call `animateStreaming(container, message.content, onAnimationDone)`. Store the returned cancel function via `chatStore.registerAnimationCancel(cancel)`.
  - `onAnimationDone`: call `highlightCodeBlocks(container)` on the streaming-markdown output, then call `chatStore.markDone(message.id)`. After markDone, swap the container innerHTML to `await renderMarkdown(message.content)` for the final sanitized render.
  - If `message.isStreaming` transitions from true to false while animation is running (user hit stop): the cancel function was already called by chatStore.stop. In a reactive $effect watching `message.isStreaming`, when it becomes false and the container exists: swap to `await renderMarkdown(message.content)` immediately (shows full response with proper formatting).
  - IMPORTANT: Each MessageBubble gets its own container element — never reuse across messages (Pitfall 2).

  **Copy buttons:**
  - Per code block: After final render (renderMarkdown swap), find all `pre > code` elements in the container. For each, inject a copy button (absolute positioned top-right of the `pre` block). The `pre` should be `relative`. On click, copy the `textContent` of the `code` element to clipboard, show brief "Copied!" feedback.
  - Whole message copy: A small copy icon button at the bottom-right of the assistant bubble (only visible on hover). On click, copy `message.content` (raw markdown) to clipboard.
  - Use `Copy` and `Check` icons from lucide-svelte.

  **Timestamps (discretion):** Show timestamp on hover via a small tooltip-style element — `new Date(message.timestamp).toLocaleTimeString()`. Use opacity transition, not a full tooltip component.

  **InputBar.svelte** — Textarea with send/stop button.

  State: `inputText = $state('')` local to component.

  Layout:
  - Container: `flex items-end gap-2 p-4 border-t border-border bg-background` pinned at bottom.
  - Content area wrapper: `max-w-3xl mx-auto w-full flex items-end gap-2` (same max-width as messages).
  - Textarea: `flex-1 min-h-[44px] max-h-[200px] overflow-y-auto resize-none rounded-xl border border-input bg-background px-4 py-3 text-sm focus:outline-none focus:ring-2 focus:ring-ring`.
  - Auto-resize via Svelte action: on `input` event, set `node.style.height = 'auto'` then `node.style.height = node.scrollHeight + 'px'`. Define as an action in the same file or inline.

  Keyboard (LOCKED — Enter sends, Shift+Enter newline):
  - `onkeydown`: if `e.key === 'Enter' && !e.shiftKey`, call `e.preventDefault()` then `handleSend()`.
  - IMPORTANT: Must preventDefault BEFORE send to avoid newline insertion (Anti-pattern from RESEARCH.md).

  Send/Stop button (LOCKED — ChatGPT-style transform):
  - Single button that swaps between Send (ArrowUp icon) and Stop (Square icon) based on `chatStore.status`.
  - Show Send when `status === 'idle'` — disabled if `inputText.trim()` is empty.
  - Show Stop when `status === 'loading' || status === 'streaming'` — always enabled.
  - On send: call `chatStore.sendMessage(inputText.trim())`, clear inputText, reset textarea height.
  - On stop: call `chatStore.stop()`.
  - Button styling: `rounded-full w-9 h-9 flex items-center justify-center bg-primary text-primary-foreground disabled:opacity-40 transition-all`.

  Message queuing (discretion decision): Disabled. Textarea remains editable for pre-typing, but Send button is disabled when status !== 'idle'. This is the simplest approach with no edge cases.
  </action>

  <done>MessageBubble renders user/assistant/error messages with two-mode markdown, code copy buttons, whole-message copy, and hover timestamps. InputBar has auto-resize textarea with Enter-to-send and send/stop button transform.</done>
