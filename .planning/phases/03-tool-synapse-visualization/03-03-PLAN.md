  <action>
Create three components for Synapse workflow visualization:

**QACard.svelte** — Props: `card: QACard` (from types.ts)
- Renders a single Q&A pair as a compact card.
- Question: `MessageCircleQuestion` icon (Lucide) + question text in `text-sm font-medium`.
- Answer: If `card.answer` exists, show below the question with a `Reply` icon (Lucide) + answer text in `text-sm text-muted-foreground`. Indent slightly to show hierarchy.
- If no answer yet, show a subtle `Loader2 animate-spin` next to the question to indicate waiting.
- Styling: `rounded-lg border border-border bg-muted/30 px-3 py-2`. Compact — these stack vertically.

**TaskChecklist.svelte** — Props: `tasks: TaskItem[]` (from types.ts)
- Renders a vertical list of tasks with status indicators.
- Each task row: status icon | description text | response preview (if done)
- Status icons (Lucide):
  - `pending`: `Circle` in `text-muted-foreground` (empty circle)
  - `running`: `Loader2` with `animate-spin` in `text-primary` (blue spinner)
  - `done`: `CheckCircle2` in `text-emerald-500` (green check)
- Description: `text-sm`. Running tasks get `font-medium` to stand out.
- Response preview: If task is done and has `responsePreview`, show it below the description in `text-xs text-muted-foreground truncate max-w-full`.
- Overall progress: Show a simple count above the list — "3 of 5 tasks complete" in `text-xs text-muted-foreground`. Calculate from tasks with status === 'done'.
- Styling: `space-y-2` for task spacing.

**SynapsePanel.svelte** — Props: `synapseState: SynapseState` (from types.ts)
- This is the main container that composes QACard and TaskChecklist.
- Renders inline in the chat stream (not a sidebar) — it IS the response for Synapse workflows.

Layout:
1. **Header row:** Synapse icon (`Workflow` from Lucide) + "Synapse Workflow" label + mode badge (`synapseState.mode` in a `rounded-full bg-primary/10 text-primary px-2 py-0.5 text-xs` pill).

2. **Phase indicator:** Three-step label showing current phase. Use a simple horizontal row of phase labels with the current one highlighted:
   - "Q&A" | "Executing" | "Complete"
   - Current phase: `text-foreground font-medium`
   - Past phases: `text-muted-foreground`
   - Future phases: `text-muted-foreground/50`
   - Use small dots or dashes between labels as separators.

3. **Q&A section:** Only visible when `synapseState.qaCards.length > 0`.
   - Render each card with `{#each synapseState.qaCards as card (card.areaId)}` using QACard component.
   - `space-y-2` between cards.

4. **Task section:** Only visible when `synapseState.tasks.length > 0`.
   - Render TaskChecklist component with `tasks={synapseState.tasks}`.

5. **Completion message:** If `synapseState.tasksCompleteMessage`, show it in `text-sm text-emerald-500 font-medium` with a `CheckCircle2` icon.

**Styling:** Outer wrapper `rounded-2xl rounded-bl-sm bg-muted px-4 py-3 max-w-[80%]` — matches the assistant bubble styling from MessageBubble. Internal sections separated by `space-y-4`.

**Cancel:** SYN-04 — cancel is handled by the EXISTING stop button in InputBar, which calls `chatStore.stop()`. The stop button already shows when `chatStore.status !== 'idle'`. No new cancel button needed inside SynapsePanel — the existing InputBar stop button covers this requirement because the Synapse workflow runs as a normal chat request (loading state).
