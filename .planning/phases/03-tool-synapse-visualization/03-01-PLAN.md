<task type="auto">
  <name>Task 2: Wire ChatStore to handle tool and Synapse SSE events</name>
  <files>web/src/lib/chat/ChatStore.svelte.ts</files>
  <action>
Extend the ChatStore class to handle all new SSE events. Key design decisions from research:

**Non-reactive instance fields to add:**
- `pendingToolCalls: ToolCall[]` — buffers tool calls during a request, attached to the assistant message on `done`
- `activeSynapseMessageId: string | null` — tracks the current synapse message for in-place updates

**Modify the `onEvent` callback inside `sendMessage`:**

For `tool` events:
- FILTER: Skip events where `d.tool` ends with `find_tool` — only show `call_tool` results. The heuristic: if `d.result` is present OR `d.hadError` is true, it's a call_tool result (find_tool only returns the tool name, call_tool returns actual results). Simpler approach: check if `d.args` is present — find_tool responses don't include args.
- Actually, simplest correct filter: the orchestrator sends TWO tool events per tool use — one for find_tool (tool='find_tool', args={name:'...'}, result='tool_name') and one for call_tool (tool='actual_tool_name', args={...actual args...}, result={...}). So filter out events where `d.tool === 'find_tool'`.
- For non-filtered tool events: create a `ToolCall` object with `status: d.result !== undefined || d.hadError ? 'done' : 'running'`, `startedAt: Date.now()`, `hadError: Boolean(d.hadError)`. If done, set `durationMs: 0` (we don't have start time for single-event tools). Push to `this.pendingToolCalls`.
- Keep the existing `this.currentActivity = 'Using: ${d.tool}'` line.

For `synapse_start` events:
- Create a new message with `role: 'synapse'`, `id: crypto.randomUUID()`, `content: ''`, `timestamp: Date.now()`, `isStreaming: false`, `synapseState: { sessionId: d.session_id, mode: d.mode, phase: 'qa', qaCards: [], tasks: [] }`.
- Push this message to `this.messages` array (use spread: `this.messages = [...this.messages, newMsg]`).
- Store the message id in `this.activeSynapseMessageId`.
- Set `this.currentActivity = 'Synapse: starting...'`.

For `synapse_question` events:
- Find the synapse message by `this.activeSynapseMessageId`.
- If found, push a new QACard `{ areaId: d.area_id, question: d.text }` to its `synapseState.qaCards`.
- Use spread to trigger reactivity: rebuild the messages array with the updated synapse message.
- Set `this.currentActivity = 'Synapse: asking questions...'`.

For `synapse_answer` events:
- Find the synapse message, find the QACard with matching `areaId`, set its `answer` property.
- Use spread to trigger reactivity.

For `task_start` events:
- Find the synapse message. Update its `synapseState.phase` to `'executing'` if not already.
- Push a new TaskItem `{ id: d.id, description: d.description, status: 'running' }` to `synapseState.tasks`.
- Mark any previously 'running' tasks as 'done' (only one task runs at a time).
- Use spread to trigger reactivity.
- Set `this.currentActivity = 'Synapse: executing tasks...'`.

For `task_done` events:
- Find the synapse message, find the TaskItem with matching `id`, set `status: 'done'` and `responsePreview: d.response`.
- Use spread to trigger reactivity.

For `tasks_complete` events:
- Find the synapse message, set `synapseState.phase = 'complete'` and `synapseState.tasksCompleteMessage = d.message`.
- Use spread to trigger reactivity.
- Set `this.currentActivity = 'Synapse: complete'`.

**Modify the `done` event handler:**
- When building the assistant message, attach `toolCalls: this.pendingToolCalls.length > 0 ? [...this.pendingToolCalls] : undefined`.
- After attaching, reset `this.pendingToolCalls = []`.
- If `this.activeSynapseMessageId` is set, also update the synapse message's phase to 'complete' (in case tasks_complete wasn't received).

**Modify `stop()` and cleanup:**
- In `stop()`, also clear `this.pendingToolCalls = []` and `this.activeSynapseMessageId = null`.
- In the `finally` block of `sendMessage`, also reset `this.pendingToolCalls = []` and `this.activeSynapseMessageId = null`.

**Helper method for synapse message updates:**
Create a private helper `updateSynapseMessage(updater: (state: SynapseState) => SynapseState)` that:
1. Finds the message with id === this.activeSynapseMessageId
2. If not found, returns
3. Applies the updater to its synapseState
4. Rebuilds messages array with spread for reactivity

This avoids repeating the find-update-spread pattern in every handler.

**IMPORTANT reactivity note:** Do NOT use push() on this.messages. Always use `this.messages = [...this.messages, newMsg]` or `this.messages = this.messages.map(...)`. The existing code already follows this pattern.
