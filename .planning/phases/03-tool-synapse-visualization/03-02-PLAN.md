  <name>Task 1: Create ToolCallBlock.svelte</name>
  <files>web/src/lib/components/ToolCallBlock.svelte</files>
  <action>
Create a collapsible tool call block component. Props: `toolCall: ToolCall`.

**Collapsed state (default):**
- Single row showing: status icon | tool name | elapsed time
- Status icons (use Lucide Svelte):
  - `running`: `Loader2` with `animate-spin` class (spinner)
  - `done`: `Check` in green (text-emerald-500)
  - `error`: `X` in red (text-destructive)
- Tool name: `text-sm font-medium text-foreground`
- Elapsed time: `text-xs text-muted-foreground` — format as `${durationMs}ms` if available, or show live elapsed counter for running status
- Clickable row — clicking toggles expanded state
- Chevron icon (ChevronRight from Lucide) that rotates 90deg when expanded
- Subtle left border for visual grouping: `border-l-2 border-border` (or `border-destructive` on error)

**Expanded state:**
- Arguments section: label "Arguments" in `text-xs text-muted-foreground`, then `<pre class="...">` with `JSON.stringify(toolCall.args, null, 2)`. Use `text-xs bg-background rounded p-2 overflow-x-auto` styling. Do NOT use {@html} — use text content only.
- Result section: label "Result" in `text-xs text-muted-foreground`, then the result.
  - Truncation: If stringified result > 300 chars, show first 300 chars with "..." and a "Show more" button.
  - "Show more" toggles a `showFullResult` state variable. When true, show full result and button text changes to "Show less".
  - Use `<pre>` for result display too — same styling as args.
- Error section: If `toolCall.hadError && toolCall.error`, show error text in `text-destructive text-xs`.
- Only show Arguments section if `toolCall.args` is defined.
- Only show Result section if `toolCall.result` is defined.

**Auto-expand on error:** If `toolCall.hadError` is true, initialize expanded state to true.

**Elapsed timer for running tools:** Use `$effect` with `setInterval(1000)` that increments a local `elapsedMs` state. Return cleanup function to clear the interval. Only run when `toolCall.status === 'running'`. When status changes to done/error, clear interval and use `toolCall.durationMs` if available.

**Styling:** Outer wrapper `rounded-lg bg-muted/50 px-3 py-2` with `cursor-pointer` on the header row. Keep it compact — this sits inside the assistant message bubble.
  </action>
