**ChatStore additions** — Add these methods to the ChatStore class:

1. `resetSession = (sessionId?: string): void` — Clears messages, sets status to idle, clears errors/activity. If sessionId provided, sets this.sessionId to it. Otherwise generates new UUID. This is used for both "new chat" and "switch session".

2. `loadSession = (sessionId: string): void` — Calls resetSession(sessionId). (Messages load naturally from the next send — sessions are server-side. The frontend just needs to point to the right session ID.)

**SessionSidebar.svelte** — Create a collapsible sidebar component:

**Props:** None (reads chatStore directly).

**State:**
- `sidebarOpen` = $state(true) — default open per discretion
- `sessions` = $state<SessionInfo[]>([]) — fetched from API
- `deletingId` = $state<string | null>(null) — ID of session showing delete confirmation
- `activeTab` = $state<'sessions' | 'knowledge'>('sessions') — tab toggle (knowledge tab built in Plan 03)

**Behavior:**
- On mount ($effect): call fetchSessions(), refresh every 30 seconds. Clean up interval on destroy.
- Group sessions by time period using a helper: "Today", "Yesterday", "Previous 7 Days", "Older". Use date comparison (not relative time formatting).
- Each session row shows: title (truncated) + relative timestamp using Intl.RelativeTimeFormat (see RESEARCH.md Pattern 5).
- Hover on a session row reveals a Trash2 icon (lucide-svelte).
- Clicking trash sets deletingId to that session. The row transforms to show "Delete?" text with a red confirm button and a cancel (X) button. No modal.
- Confirm delete: call deleteSession(id), remove from local sessions array, clear deletingId.
- If deleted session was the active one (chatStore.sessionId), call chatStore.resetSession() to start fresh.
- "New chat" button at top: Plus icon + "New chat" text. Calls chatStore.resetSession().
- Clicking a session row: calls chatStore.loadSession(session.id). Highlight active session (chatStore.sessionId match).

**Collapse behavior:**
- Toggle button: PanelLeftClose / PanelLeftOpen icon from lucide-svelte, placed at top of sidebar.
- Use CSS max-width transition per RESEARCH.md Pattern 2: `transition-[max-width] duration-200 ease-in-out overflow-hidden`. max-width 256px when open, 0px when closed.
- Keep aside in DOM always (never use {#if}). Use overflow-hidden to hide content when collapsed.

**Tab toggle:**
- Two tabs at top of sidebar below the toggle: "Sessions" and "Knowledge".
- Use simple button styling with border-bottom highlight for active tab.
- When activeTab is 'knowledge', show a placeholder div with text "Knowledge base — coming soon" (Plan 03 will replace this).

**Styling:** Tailwind dark theme classes consistent with existing components. border-r border-border, bg-background.

**HealthBar.svelte** — Full-width top bar above the chat area:

**State:**
- `health` = $state<HealthStatus | null>(null)
- `modelName` = $state<string>('Loading...')

**Behavior:**
- On mount ($effect): call fetchHealth() and fetchModel(). Poll both every 30 seconds. Clean up interval on destroy.
- Parse health status to determine per-service health:
  - LM Studio: healthy if `health.errors` does not contain any string matching "LM Studio"
  - RAG: healthy if `health.errors` does not contain any string matching "RAG"
  - Search: For v1, derive from RAG health (if RAG is healthy, Search is healthy). The chimera-chat.js health endpoint doesn't separate RAG and Search, so treat them together. Show "RAG + Search" as one indicator.
- Actually, re-reading the research: chimera-chat.js /api/health?deep=true returns `{ status, errors[] }` where errors are strings like "RAG stack unreachable" and "LM Studio unreachable". So parse these strings.

**Layout (left to right):**
- Service indicators: For each service (LM Studio, RAG + Search), show:
  - Green dot (bg-green-500 rounded-full w-2 h-2) + service name when healthy
  - Red dot (bg-red-500) + service name + inline error text (e.g., "offline") when unhealthy
  - Use `transition-colors duration-300` on the dots per discretion
- Model name: text-sm text-muted-foreground, shows modelName value
- Right side: "Local" badge — small pill with Shield icon (lucide-svelte) + "Local" text, bg-green-500/10 text-green-500 rounded-full px-2 py-0.5 text-xs

**Styling:** flex items-center gap-4, border-b border-border, px-4 py-1.5, text-xs. Compact — should not take much vertical space.
