**KnowledgeSidebar.svelte** — Document management panel rendered inside the sidebar:

**State:**
- `documents` = $state<KnowledgeDocument[]>([]) — from fetchDocuments()
- `searchQuery` = $state('') — client-side filter
- `filteredDocs` = $derived — filter documents by searchQuery matching filename (case-insensitive)
- `dragCount` = $state(0) — drag-and-drop counter (RESEARCH.md Pattern 3 — use counter, not boolean, to avoid flicker from event bubbling)
- `isDragging` = $derived(dragCount > 0)
- `uploading` = $state(false) — shows indeterminate pulse during upload
- `deletingId` = $state<string | null>(null) — inline delete confirmation (same pattern as SessionSidebar)

**On mount ($effect):** Fetch documents. No polling needed (documents don't change externally often). Refresh after upload/delete.

**Layout (top to bottom):**
1. Search input: text input with Search icon (lucide-svelte), placeholder "Filter documents...", filters the list client-side.
2. Drop zone / document list area:
   - When isDragging: show a dashed border overlay with Upload icon and "Drop to upload" text. Use `bg-primary/5 border-2 border-dashed border-primary rounded-lg` styling.
   - When not dragging: show the document list.
3. Document list: Each row shows:
   - FileText icon (lucide-svelte) as type indicator
   - Filename (truncated)
   - Date (formatted as short date, e.g., "Mar 12")
   - Hover reveals Trash2 icon for delete
   - Delete confirmation: same inline pattern as SessionSidebar (row transforms to "Delete?" + confirm + cancel)
4. Upload state: When uploading is true, show an indeterminate pulse bar at the top of the list area (a thin div with `animate-pulse bg-primary/50 h-0.5`).
5. Empty state: When no documents and no search query, show "No documents yet" + "Drag files here to upload" text centered.

**Drag-and-drop handlers** on the sidebar content area (NOT full window — sidebar-only per discretion):
- `ondragenter`: increment dragCount, prevent default
- `ondragleave`: decrement dragCount
- `ondragover`: prevent default (required to enable drop)
- `ondrop`: prevent default, reset dragCount to 0, extract files from e.dataTransfer.files, for each file call uploadDocument(file). Set uploading=true, await all uploads, set uploading=false, re-fetch documents.
- Only accept the first file if multiple dropped (keep it simple for v1).

**Also add a click-to-upload fallback:** Small "Upload" button (Upload icon) that triggers a hidden `<input type="file">`.
