<task type="auto">
  <name>Task 1: Scaffold SvelteKit project with Tailwind v4 and shadcn-svelte</name>
  <files>web/ (entire new directory)</files>
  <action>
Run the scaffold commands from the project root (C:/Users/sandv/Desktop/chimera):

1. Create the SvelteKit project:
```bash
npx sv create web --template minimal --types ts --no-add-ons --no-install
```
Note: We use --no-add-ons because we'll configure Tailwind v4 manually to ensure the Vite plugin approach (not PostCSS). Run --no-install so we can add shadcn-svelte deps before installing.

2. cd into web/ and install adapter-static:
```bash
cd web
npm install --save-dev @sveltejs/adapter-static
```

3. Install Tailwind v4 with Vite plugin:
```bash
npm install --save-dev tailwindcss @tailwindcss/vite
```

4. Initialize shadcn-svelte (this adds bits-ui, clsx, tailwind-merge, tw-animate-css, lucide-svelte, and creates components.json + cn() utility):
```bash
npx shadcn-svelte@latest init
```
When prompted, accept defaults. If it asks about style, choose "default". If it asks about base color, choose "zinc". If it asks about CSS variables, yes.

5. Install all dependencies:
```bash
npm install
```

After scaffolding, verify and fix these files:

**web/svelte.config.js** — Must use adapter-static with fallback: '200.html':
```javascript
import adapter from '@sveltejs/adapter-static';

/** @type {import('@sveltejs/kit').Config} */
const config = {
  kit: {
    adapter: adapter({
      fallback: '200.html'
    })
  }
};

export default config;
```

**web/vite.config.ts** — Must use @tailwindcss/vite plugin AND /api proxy:
```typescript
import { sveltekit } from '@sveltejs/kit/vite';
import { defineConfig } from 'vite';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
  plugins: [tailwindcss(), sveltekit()],
  server: {
    proxy: {
      '/api': {
        target: 'http://127.0.0.1:3210',
        changeOrigin: true
      }
    }
  }
});
```

**web/src/routes/+layout.ts** — Disable SSR globally for SPA mode:
```typescript
export const ssr = false;
```

**web/src/routes/+layout.svelte** — Svelte 5 syntax with app.css import:
```svelte
<script lang="ts">
  import '../app.css';
  let { children } = $props();
</script>

{@render children()}
```

**web/src/app.css** — Tailwind v4 import (shadcn-svelte init may have already set this up; verify it uses `@import "tailwindcss"` NOT the three-directive pattern):
```css
@import "tailwindcss";
```
Keep any shadcn-svelte theme variables that the init command added (like @custom-variant dark, @theme blocks, etc).

Delete postcss.config.js if it exists — Tailwind v4 uses the Vite plugin only.

Delete tailwind.config.js or tailwind.config.ts if it exists — Tailwind v4 uses CSS @theme.
